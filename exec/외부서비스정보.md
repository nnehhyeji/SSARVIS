# SSARVIS 외부 서비스 정보 정리

## 1. 문서 목적

- 이 문서는 프로젝트에서 사용하는 외부 서비스의 가입 대상, 준비 정보, 실제 사용 위치를 정리한 문서다.
- 민감한 키/토큰 값은 여기에 직접 적지 않고, 변수명과 관리 위치만 기록한다.

## 2. 외부 서비스 목록

| 서비스 | 용도 | 준비 항목 | 주요 설정 변수/파일 |
| --- | --- | --- | --- |
| OpenAI 호환 API Gateway | LLM 응답 생성, 임베딩 생성 | API Key, base URL, 허용 모델명 | `OPENAI_API_KEY`, `OPENAI_BASE_URL`, `LLM_MODEL`, `EMBEDDING_MODEL` |
| DashScope | 음성 등록, 실시간 TTS | API Key, 사용 모델, WebSocket/HTTP 엔드포인트 | `DASHSCOPE_API_KEY`, `TTS_MODEL`, `TTS_WS_URL`, `TTS_ENROLLMENT_URL` |
| Kakao Developers | 소셜 로그인 | REST API Key, Client Secret, Redirect URI 등록 | `KAKAO_CLIENT_ID`, `KAKAO_CLIENT_SECRET`, `KAKAO_REDIRECT_URI`, `VITE_KAKAO_*` |
| AWS S3 | 파일 업로드 저장소 | 버킷, IAM Access Key, Region | `S3_BUCKET_NAME`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION` |
| AWS CloudFront | 업로드 파일 CDN 제공 | 배포 도메인, S3 origin 연동 | `CLOUDFRONT_DOMAIN` |
| Gmail SMTP | 메일 발송 | SMTP 계정, 앱 비밀번호 | `MAIL_USERNAME`, `MAIL_PASSWORD` |
| Slack Webhook | 슬로우 쿼리/장애 알림 | Incoming Webhook URL | `SLACK_WEBHOOK_URL` |
| Mattermost Webhook | 서비스 알림/스케줄 알림 | Incoming Webhook URL | `MATTERMOST_WEBHOOK_URL` |
| Docker Hub 또는 이미지 레지스트리 | 운영 이미지 pull | 계정, access token, image/tag 관리 | `DOCKERHUB_USERNAME`, `DOCKERHUB_TOKEN`, `*_IMAGE`, `*_TAG` |
| SSL 인증서 발급 체계 | HTTPS 제공 | 도메인, 인증서, 개인키, certbot challenge 경로 | `infra/nginx/conf.d/default.conf`, cert 파일 경로 |

## 3. 서비스별 상세

### 3.1 OpenAI 호환 API Gateway

용도

- AI 서버에서 텍스트 응답 생성
- 사용자/대화 임베딩 생성

가입 및 준비 정보

- OpenAI 또는 OpenAI 호환 게이트웨이 계정
- API Key 발급
- 사용할 base URL 확인
- 사용할 모델명과 임베딩 차원 합의

프로젝트 반영 위치

- `ai/app/infra/openai.py`
- `ai/app/config/openai.py`
- `ai/.env.example`
- 운영 시 `infra/deploy/env/ai.env`, `infra/deploy/env/ai.secret.env`

주의사항

- `EMBEDDING_DIMENSIONS` 와 `QDRANT_VECTOR_SIZE` 가 반드시 같아야 한다.
- 예시 파일과 코드 기본 모델값이 다를 수 있으므로 실배포 전 통일 필요.

### 3.2 DashScope

용도

- 사용자 음성 등록
- 등록된 음성 기반 실시간 TTS

가입 및 준비 정보

- DashScope 계정 생성
- API Key 발급
- 사용 모델별 이용 가능 여부 확인
- 서비스 지역/요금제/트래픽 제한 확인

프로젝트 반영 위치

- `ai/app/infra/dashscope.py`
- `ai/app/domains/voice/service.py`
- `ai/app/config/dashscope.py`
- `ai/.env.example`

주의사항

- 음성 등록 시 오디오를 MP3/base64 형태로 전송한다.
- 실시간 TTS는 WebSocket 연결이 필요하다.

### 3.3 Kakao Developers

용도

- 사용자 소셜 로그인

가입 및 준비 정보

- Kakao Developers 애플리케이션 생성
- REST API Key 확보
- Client Secret 사용 설정
- Redirect URI 등록
- 동의 항목에서 이메일 제공 허용

등록해야 하는 URI 예시

- Backend callback 예시: `http://localhost:8080/api/v1/auth/kakao/callback`
- Frontend callback 페이지 예시: `http://localhost:5173/auth/kakao/callback`

프로젝트 반영 위치

- Backend
  - `backend/src/main/resources/application.yaml`
  - `backend/src/main/java/com/ssafy/ssarvis/auth/service/OAuthService.java`
  - `backend/src/main/java/com/ssafy/ssarvis/auth/controller/AuthController.java`
- Frontend
  - `frontend/src/pages/auth/SignupPage.tsx`
  - `frontend/src/pages/auth/LoginPage.tsx`
  - `frontend/src/pages/auth/KakaoCallbackPage.tsx`

주의사항

- 프런트와 백엔드가 서로 다른 redirect URI 를 사용하므로 둘 다 Kakao 콘솔/시스템 설정과 일치해야 한다.
- 사용자 이메일 동의가 없으면 로그인 흐름이 실패할 수 있다.

### 3.4 AWS S3 / CloudFront

용도

- 사용자 업로드 파일 저장
- 저장 파일을 CloudFront URL 로 제공

가입 및 준비 정보

- AWS 계정
- S3 버킷 생성
- IAM 사용자 또는 역할 생성
- Access Key/Secret 발급
- CloudFront 배포 생성 및 S3 연결
- CORS/버킷 정책/퍼블릭 접근 정책 검토

프로젝트 반영 위치

- `backend/src/main/java/com/ssafy/ssarvis/common/config/S3Config.java`
- `backend/src/main/java/com/ssafy/ssarvis/common/service/S3Uploader.java`
- `backend/src/main/resources/application.yaml`

주의사항

- 프로젝트는 S3 직접 URL 대신 CloudFront URL 을 반환한다.
- CloudFront 도메인이 바뀌면 기존 URL 처리 로직에도 영향이 있다.

### 3.5 Gmail SMTP

용도

- 인증 메일 또는 일반 메일 발송

가입 및 준비 정보

- Gmail 계정
- 2단계 인증 활성화
- 앱 비밀번호 발급

프로젝트 반영 위치

- `backend/src/main/resources/application.yaml`
- `compose-front/.env`

주의사항

- 일반 계정 비밀번호로는 동작하지 않을 수 있다.
- 보안상 앱 비밀번호는 secret env 파일로만 관리한다.

### 3.6 Slack / Mattermost Webhook

용도

- 슬로우 쿼리 알림
- 배치/운영 알림

가입 및 준비 정보

- Slack Workspace 또는 Mattermost 채널 접근 권한
- Incoming Webhook URL 생성

프로젝트 반영 위치

- `backend/src/main/resources/application.yaml`
- `backend/src/main/java/com/ssafy/ssarvis/common/service/SlackNotificationService.java`
- `backend/src/main/java/com/ssafy/ssarvis/common/service/MattermostService.java`

주의사항

- webhook URL 유출 시 외부 스팸 전송 위험이 있으므로 secret 파일로만 관리한다.

### 3.7 Docker Hub / 컨테이너 레지스트리

용도

- 운영 배포 시 애플리케이션 이미지 pull

가입 및 준비 정보

- Docker Hub 계정 또는 사설 레지스트리 계정
- access token 발급
- `backend`, `ai`, `web` 이미지 push 권한

프로젝트 반영 위치

- `infra/deploy/scripts/deploy.sh`
- `compose-front/.env`
- 운영 `infra/deploy/env/app.env`, `infra/deploy/env/ai.env`

주의사항

- 운영 배포 스크립트는 `docker compose pull` 중심이라 이미지가 registry 에 올라가 있어야 가장 자연스럽다.

## 4. 외부 서비스 가입 체크리스트

### 4.1 필수

- Docker / Docker Compose 실행 가능 서버
- Kakao Developers 앱
- OpenAI 호환 API Key
- DashScope API Key
- AWS S3 + CloudFront
- Gmail SMTP 앱 비밀번호

### 4.2 운영 품질 향상용

- Slack Webhook
- Mattermost Webhook
- Docker Hub access token

## 5. 민감 정보 관리 원칙

- `.env`, `*.secret.env` 에 실제 값을 저장한다.
- Git 저장소에는 예시 파일과 변수명만 남긴다.
- 문서에는 절대 실제 토큰/비밀번호/secret 값을 적지 않는다.
- 운영 서버의 `infra/deploy/env/*.env` 파일은 접근 권한을 제한한다.

## 6. 확인이 필요한 항목

- OpenAI base URL 이 일반 OpenAI인지, 사내 게이트웨이인지 운영 환경별로 다를 수 있다.
- Kakao 로그인에서 REST API Key 와 실제 앱 설정이 프런트/백엔드 양쪽 URI 와 일치하는지 배포 전 확인이 필요하다.
- AWS 권한 범위는 최소 권한 원칙으로 재설계하는 것이 좋다.
- Slack/Mattermost Webhook 은 운영 채널 정책에 따라 교체될 수 있다.
